## 1. Как сдать работу на рецензию

### Шаг 1. Подготовьте файл с теорией

Скопируйте теоретическую часть вашего ДЗ в файл:

```
reviews/{ваш_github_login}/theory.md
```

Например, если ваш логин `alice`:

```
reviews/alice/theory.md
```

Формулы оформляйте в LaTeX. Inline: `$f(x) = x^2$`. Блочные:

```
$$
\frac{\partial L}{\partial w} = \sum_{i=1}^{n} \delta_i x_i
$$
```

### Шаг 2. Запушьте ветку

Ветка должна называться `{ваш_github_login}/{номер_дз}`, например `alice/hw2`:

```bash
git clone https://github.com/<org>/peer-review-hub
cd peer-review-hub
git checkout -b alice/hw2
mkdir -p reviews/alice
# скопируйте ваш theory.md в reviews/alice/theory.md
git add reviews/alice/theory.md
git commit -m "hw2: theory"
git push origin alice/hw2
```

### Шаг 3. Откройте Pull Request

1. Перейдите в репозиторий `peer-review-hub` на GitHub
2. Нажмите **Compare & pull request** (появится после пуша ветки)
3. Убедитесь, что:
   - **base:** `main`
   - **compare:** `alice/hw2` (ваша ветка)
4. Заполните чеклист в описании PR
5. Нажмите **Create pull request**

> ⚠️ Не открывайте PR раньше, чем закончите теоретическую часть — после открытия PR рецензент назначается автоматически.

### Шаг 4. Дождитесь рецензента

В течение нескольких минут бот оставит комментарий с именем назначенного рецензента. Если сдавших пока мало, второй рецензент будет назначен позже — вы получите уведомление от GitHub.

## 2. Как сделать рецензию

Вам придёт уведомление от GitHub когда вас назначат рецензентом.

### Шаг 1. Откройте PR

Перейдите по ссылке из уведомления или найдите PR в репозитории `peer-review-hub`.

### Шаг 2. Изучите работу

Перейдите на вкладку **Files changed** — там виден файл `theory.md` автора.

### Шаг 3. Оставьте inline-комментарии

Наведите курсор на строку → нажмите **+** слева → напишите комментарий → **Start a review**.

Требования к комментариям:
- **Минимум 2 inline-комментария** к конкретным строкам файла
- Комментарии должны быть содержательными: указывайте на ошибки, неточности, или задавайте уточняющие вопросы
- Если всё верно — напишите почему вы согласны с формулировкой

Примеры хороших комментариев:
> «Формула для KL-дивергенции верна, но не хватает пояснения, почему результат всегда ≥ 0»

> «Здесь перепутаны индексы i и j в сумме — проверьте вывод»

> «Объяснение reparameterization trick корректное и понятное»

### Шаг 4. Завершите review

1. Нажмите кнопку **Review changes** (верхний правый угол вкладки Files changed)
2. Напишите общий итог рецензии (1–3 предложения)
3. Выберите один из вариантов:
   - **Approve** — работа верна, замечаний нет или они незначительные
   - **Request changes** — есть ошибки, которые нужно исправить
4. Нажмите **Submit review**

> ⚠️ Рецензия не засчитывается если вы только написали комментарии, но не нажали **Submit review**.

## 3. Требования для получения оценки

Peer review засчитывается автоматически при выполнении всех условий:

| Условие | Как проверяется |
|---|---|
| Вы сдали теоретическую часть (открыли PR) | PR существует |
| Ваша работа получила 2 рецензии | Бот ставит лейбл `peer-review-complete` |
| Вы сами проверили 2 чужие работы | Счётчик в `state/hw*.json` |
| Каждая ваша рецензия: formal review (Approve / Request changes) | GitHub API |
| Каждая ваша рецензия: минимум 2 inline-комментария | GitHub API |

Когда все условия выполнены, бот поставит лейбл **`peer-review-complete`** на ваш PR и напишет комментарий.

## 4. Частые вопросы

**Я открыл PR, но рецензент не назначен — почему?**

Вы сдали первым или одним из первых. Рецензент будет назначен автоматически, когда следующий студент откроет свой PR. Ждать ничего не нужно.

**Могу ли я видеть чужие PR?**

Да — все PR в `peer-review-hub` видны участникам организации. Но смотреть и копировать чужие ответы бессмысленно: цель peer review — разобраться самому и помочь другому.

**Рецензент попросил исправления (Request changes) — что делать?**

Внесите правки в `reviews/{ваш_login}/theory.md`, запушьте в ту же ветку. PR обновится автоматически, рецензент получит уведомление.

**Прошло 3 дня, а рецензент молчит — что делать?**

Бот автоматически напомнит рецензенту. Если через ещё 2 дня ничего — напишите преподавателю.

**Я случайно закрыл PR — что делать?**

Переоткройте PR или напишите преподавателю.
